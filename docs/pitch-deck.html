Agentic AI Dev · 06 / 10
How We Built It

Agentic AI ile Hızlandırılmış Geliştirme

Cursor IDE ekosistemi, her adımda kodu bağlamı anlayan bir agent olarak yazdı. Biz mimari kararlar aldık; AI uyguladı.

📐

Cursor Ruleset (AGENTS.md)

Her modül için ayrı ruleset tanımlandı. Agent, kod yazarken bu kısıtları otomatik uygulayarak Clean Architecture'ı korudu.

# RoadScore Ruleset
- Handlers → /internal/handlers
- HF timeout: 10s hard limit
- Rate limit middleware: zorunlu
- Coords: PostGIS formatı
- Error wrap: fmt.Errorf("%w")
🔁

Cursor CLI & SDK Entegrasyonu

Tekrarlayan Go boilerplate üretimi, API handler iskeletleri ve test senaryoları Cursor CLI komutlarıyla otomatize edildi.

# Handler iskelet üretimi
cursor run generate-handler \
  --module roadscore \
  --pattern clean-arch

# SDK ile otomatik entegrasyon testi

🤖 Agent Çalışma Döngüsü

1
Mimari Prompt — "Go handler yaz, masterfabric-go Clean Architecture + bu ruleset" → Agent tüm layer'ları oluşturdu
2
Compose Mode — HF API entegrasyonu + retry logic + timeout → tek promptta çalışan pipeline
3
Context Window — Tüm codebase bağlamında KVKK kurallarını otomatik uygulayarak blur middleware ekledi
4
Hata Döngüsü — Linter hatası → Agent kendi düzeltti, commit önerdi, açıkladı
5
Paralel Geliştirme — 3 modül eş zamanlı; her modül için ayrı agent context penceresi
